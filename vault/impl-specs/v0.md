# Sweep v0 — implementation spec

> Build the spine. `sweep "<curl … | sh>"` parses the invocation, fetches the script, saves it content-addressed, executes it with original `curl | sh` semantics, and records the result in SQLite. `sweep list` reads SQLite and renders installed packages. Everything else (LLM, signals, registry, TUI, more verbs) is deferred to v1 and explicitly slotted into named seams.

**Status:** Ready for implementation.

---

## Glossary

- **Sweep home** — `~/.sweep/` (override: `$SWEEP_HOME`). The on-disk root for everything sweep persists: the DB, fetched scripts, future config and downloaded artifacts.
- **Script store** — `~/.sweep/cache/scripts/<sha256>`. Content-addressed cache of every script sweep has fetched. Identical scripts dedupe to the same file.
- **DB** — `~/.sweep/sweep.db`. SQLite database holding `packages` and `invocations` tables.
- **Install command** — the parsed form of the user-supplied `curl | sh` line: env vars, sudo flag, shell, script args, URL. Represented by the `InstallCommand` type. (Distinct from a "subcommand" — install commands are user input; subcommands are sweep verbs like `list`.)
- **Invocation** — each time the user runs `sweep "<cmd>"`. One row in the `invocations` table, regardless of outcome. (`sweep list`, unknown verbs, and other non-install runs do not produce invocation rows.)
- **Package** — one row in the `packages` table. Materializes the moment a fetch succeeds. Carries lifecycle status (`attempting | installed | failed | uninstalled | tracked`).
- **Install** — an invocation that ended with `exit_code === 0`. Promotes the package's `status` to `installed`.
- **Slug** — a human-readable identifier for a package. v0: derived from the URL host stem. v1: registry-canonical.
- **Runner shell** — the shell that executes the script (`sh`, `bash`, `zsh`). Extracted from the right side of the pipe in the user's input.
- **Fetcher** — the tool the user typed to download the script (`curl`, `wget`). Sweep ignores it at exec time; sweep fetches itself.

## Scope

**In v0:**

- Single positional arg form: `sweep "<curl … | sh>"`.
- `sweep list`.
- Idempotent first-run setup: `ensureSweepHome()`, `ensureConfig()` (v0 stub), `ensureDb()` (lazy, called from the store layer).
- Parser: 5-field extraction (`env_vars`, `sudo`, `shell`, `script_args`, `url`). Refuses chained/conditional shells.
- HTTP GET of the script URL, sha256 hash, content-addressed store.
- Exec via `Bun.spawn` with script bytes on stdin (preserves `curl | sh` stdin semantics).
- Stdout/stderr passthrough (inherited from child).
- SQLite persistence via `bun:sqlite`. Single DB file at `~/.sweep/sweep.db`.

**Explicitly out of v0:**

- LLM analysis of any kind.
- Deterministic signals (WHOIS, CT, hash DBs, sigstore).
- Recursive script fetching.
- Binary fetching/hashing.
- Centralized registry (canonical slugs, URL evolution tracking, cached verdicts).
- TUI install dialog.
- Confirm-before-exec prompt (any kind).
- First-run wizard (the `ensureConfig` seam exists but is a no-op stub).
- `--help` and `--version` flags. (Unknown flags pass through and get rejected by the parser.)
- Other subcommands: `update`, `away`, `show`, `track`, `from-history`, `--auto`.
- Bare `sweep` (no-arg interactive paste mode).
- LLM-extracted parser (deferred — v0's hand-rolled parser is behind the same interface).
- Config file / env-driven settings (other than `SWEEP_HOME`).

## Architecture

### Module layout

```
src/
  main.ts                     # orchestrator — ensure setup, dispatch verb or install
  fs.ts                       # sweepFs singleton + ensureSweepHome()
  config.ts                   # ensureConfig() — v0 stub; v1 reads ~/.sweep/config + runs wizard

  subcommands/
    types.ts                  # Subcommand interface
    dispatch.ts               # route to a subcommand handler
    registry.ts               # commands[] — install, list
    install.ts                # `sweep "<cmd>"` — thin verb → installer
    list.ts                   # `sweep list`

  installer/
    install.ts                # runInstall(raw) — the install orchestrator
    parse.ts                  # parseInstallCommand(raw) → InstallCommand | ParseError
    fetch.ts                  # fetchScript(url) → FetchedScript
    exec.ts                   # buildExecArgv + runExec

  store/
    db.ts                     # ensureDb() — bun:sqlite handle, mkdir + migrations
    packages.ts               # packages table CRUD
    invocations.ts            # invocations table CRUD
    scripts.ts                # CAS: saveScript(sha, bytes), readScript(sha), scriptPath(sha)

  identity/
    naming.ts                 # v0: slugFromUrl(url). v1: registry-backed.

  types.ts                    # shared types: InstallCommand, FetchedScript, Invocation, PackageRow, Outcome
```

## Modules

### `src/fs.ts`

Mirrors `wrap/src/fs/home.ts` plus the first-run ensure:

```ts
import { mkdirSync } from "node:fs";
import { createAppFs } from "wrap-core/fs";

export const sweepFs = createAppFs({ app: "sweep" });

/** Idempotent. Creates ~/.sweep/ if missing. Safe to call every invocation. */
export function ensureSweepHome(): void {
  mkdirSync(sweepFs.root, { recursive: true });
}
```

Process-wide singleton. All on-disk reads/writes route through it. Tests construct a fresh `AppFs` against a temp `SWEEP_HOME` (set in `bunfig.toml` preload) — never mutate env at the top of a test file.

### `src/config.ts`

v0 stub for the first-run/config seam. Mirrors wrap's `ensureConfig` pattern. Body is intentionally trivial — v1 will read `~/.sweep/config.json`, run the first-run wizard if missing, and return real settings. The signature is the stable seam; callers don't change in v1.

```ts
export type Config = {
  // empty in v0. v1 adds: llmProvider, llmKey, autoApprove, registry overrides, etc.
};

export function ensureConfig(): Config {
  return {};
}
```

### `src/main.ts`

Top-level orchestrator. Sequence:

1. `ensureSweepHome()` — mkdir the home dir.
2. `ensureConfig()` — v0 returns `{}`; v1 reads config / runs wizard.
3. Parse argv. Take the first positional arg (`process.argv[2]`, or `""` if absent).
4. If it exactly matches a verb in the subcommand registry (v0: only `list`), dispatch with the remaining positional args: `process.exitCode = await verb.run(rest)`.
5. Otherwise treat the whole first positional arg as the install command and call `runInstall(positional)`. (Empty string flows through to parse as a `ParseError` of kind `empty` — that's how `sweep` with no args fails fast.)
6. Top-level try/catch: print errors via stderr, set `process.exitCode = 1`.

No `--help` or `--version` handling in v0. Unknown flags fall through to step 5 and get rejected by the parser. (Sweep diverges from wrap here: wrap has no positional verbs and uses flag-style dispatch; sweep's package-manager surface needs positional verbs.)

The `ensureDb()` call lives inside the store modules that need it (lazy) — `main.ts` does not pre-init the database, because `sweep list` and `sweep "<cmd>"` both reach for the DB through the store layer anyway.

### `src/subcommands/`

Mirror wrap's pattern where it fits:

```ts
// types.ts
export type Subcommand = {
  name: string;
  description: string;
  run: (argv: string[]) => Promise<number>;  // returns exit code
};

// registry.ts
export const commands: Subcommand[] = [listCmd];   // installCmd is the implicit fallback in main.ts, not a registered verb

// dispatch.ts — used only from main.ts when an exact verb match is found
export async function dispatch(name: string, argv: string[]): Promise<number> {
  const cmd = commands.find(c => c.name === name);
  if (!cmd) throw new Error(`unknown verb: ${name}`);  // unreachable: main.ts only calls dispatch on matches
  return cmd.run(argv);
}
```

`list.ts` is thin: `listCmd.run([])` calls `listInstalledPackages()` and renders. `install.ts` is a thin file too, but it isn't registered in `commands` — it's wired directly from `main.ts`'s fallback branch as `runInstall(positional)`. Treating install as a verb in the registry would require carving out a special case for the `sweep "<cmd>"` shape (no verb keyword); the fallback wiring keeps `commands` clean.

### `src/installer/parse.ts`

Hand-rolled, deterministic. Accepts the universe of shapes that compress to `(env=val …) (sudo?) <shell> [-s --] [args]` with bytes coming from `<url>`. Refuses anything else.

```ts
export type InstallCommand = {
  envVars: Record<string, string>;
  sudo: boolean;
  shell: "sh" | "bash" | "zsh";        // extend later if needed
  scriptArgs: string[];
  url: string;
  raw: string;                          // original input verbatim, for logging
};

export type ParseError = {
  kind: "empty" | "chain" | "no-fetcher" | "no-pipe" | "no-url" | "unsupported";
  message: string;                      // user-facing
};

export function parseInstallCommand(input: string): InstallCommand | ParseError;
```

**Accepted shapes** (normalize to a `InstallCommand`):

| Input shape | Notes |
| --- | --- |
| `curl <flags> <url> \| sh` | Baseline. |
| `wget -qO- <url> \| sh` | Fetcher is `wget`; sweep doesn't care, just extracts the URL. |
| `curl … \| bash -s -- <args>` | `-s` is dropped at exec time (we file-mode the script). `--` and following args become `scriptArgs`. |
| `curl … \| sudo bash` | `sudo === true`. |
| `sudo curl … \| bash` | `sudo === false`. Sudo on the fetcher is moot — we re-fetch ourselves. |
| `VERSION=1.4 curl … \| sh` | `envVars = { VERSION: "1.4" }`. |
| `bash -c "$(curl <url>)"` | Normalize: `shell=bash, url=<url>`. |
| `bash <(curl <url>)` | Same as above. |

**Refused shapes** (v0 returns `ParseError`):

| Shape | `kind` |
| --- | --- |
| `""`, whitespace only | `empty` |
| `curl … && echo done`, `curl … ; sudo …`, `curl … \|\| true` | `chain` |
| `cat /tmp/x \| sh` (no fetcher, no URL) | `no-fetcher` |
| `curl <url>` (no pipe to shell) | `no-pipe` |
| `curl \| sh` (no URL) | `no-url` |
| Anything else the parser can't confidently extract from | `unsupported` |

**Implementation note:** Use plain string scanning, not a shell-grammar library. The set of accepted shapes is small and finite — recognize each pattern directly. Resist building a real shell parser. If a user pastes something exotic, refusing with a clear error is better than mis-parsing. Have a subagent write many, many TDD tests for all possible shapes before you start.

### `src/installer/fetch.ts`

```ts
export type FetchedScript = {
  bytes: Uint8Array;
  sha256: string;
  finalUrl: string;            // after redirects
  fetchedAt: string;           // ISO 8601 UTC
  status: number;              // HTTP status
};

export async function fetchScript(url: string): Promise<FetchedScript>;
```

- Uses Bun's built-in `fetch`. Follow redirects (default).
- Reasonable timeout (30s).
- Reject non-2xx with a typed error (the installer turns this into an `outcome: "fetch_failed"` invocation).
- Compute sha256 from the response bytes using Bun's crypto (`Bun.CryptoHasher`).
- Limit body size (e.g. 5 MiB). Install scripts are tiny; an unbounded read is a footgun.

### `src/installer/exec.ts`

```ts
export async function runScript(
  cmd: InstallCommand,
  scriptBytes: Uint8Array,
): Promise<{ exitCode: number }>;
```

**Exec recipe** (built inside `runScript`). `argv = [...(cmd.sudo ? ["sudo"] : []), cmd.shell, "-s", "--", ...cmd.scriptArgs]`. Always include `-s --` — harmless when there are no args and required when there are. `env` is `{ ...process.env, ...cmd.envVars }` so user-supplied env vars override the parent process's.

**Implementation.** `Bun.spawn({ cmd: argv, env, stdin: scriptBytes, stdout: "inherit", stderr: "inherit" })`, then `await child.exited; return { exitCode: child.exitCode ?? -1 }`. Bun pipes the bytes into the child's stdin — semantically identical to `cat /local/script | <argv>` but without an intermediate process.

**stdin semantics.** Script body arrives via stdin; the script's stdin is the byte stream, not the user's terminal. This matches `curl | sh` so installers that grab `/dev/tty` for prompts (ollama, nvm) continue to work identically.

**Failure surfacing.** Non-zero exit codes are returned, not thrown. The installer records them as `outcome: "errored"`. Sweep itself exits with the child's exit code (passthrough).

### `src/installer/install.ts`

The orchestrator. Single exported function:

```ts
export async function runInstall(raw: string): Promise<number>;
```

Returns the process exit code that `sweep "<cmd>"` should terminate with. Logic:

1. Parse. On `ParseError`: insert one invocation row with `outcome: "parse_failed"`, `package_id: null`, `raw_input: raw`, `ts_started`/`ts_finished` = now, `error_message: error.message`, all other columns null. Print error to stderr. Return 2.
2. Fetch. On error (network failure, non-2xx, timeout, or body-size-limit exceeded — all map to `fetch_failed`): insert invocation with `outcome: "fetch_failed"`, `package_id: null`, `raw_input: raw`, `url: parsed.url`, `install_command_json: JSON.stringify(parsed)`, `ts_started`/`ts_finished` = now, `error_message: …`. Print error to stderr. Return 1.
3. Save script. `saveScript(fetched.sha256, fetched.bytes)` — idempotent CAS write. Returns absolute path. (Path is not needed for exec since we pass bytes directly, but `saveScript` still persists the bytes for v1 `show` / diff features.)
4. Resolve package. `findOrCreatePackage({ url: parsed.url, slug: slugFromUrl(parsed.url) })`. Returns `package.id`. `status` starts as `attempting` if it's a brand new row; otherwise unchanged.
5. **v1 seam:** analysis + dialog. v0 is a no-op — proceed directly.
6. Run exec. `const { exitCode } = await runScript(parsed, fetched.bytes);`
7. Wrap both writes in `ensureDb().transaction(() => { ... })`:
   - Insert invocation: `outcome: exitCode === 0 ? "ran" : "errored"`, `package_id`, `ts_started`, `ts_finished = now`, `raw_input = raw`, `url = parsed.url`, `sha256 = fetched.sha256`, `install_command_json = JSON.stringify(parsed)`, `exit_code = exitCode`.
   - Call `updatePackageOnExec({ packageId, sha256, exitCode, ranAt: now })`, which encodes the status-transition rules: if `exitCode === 0`, set `status='installed'`, `current_sha256=sha256`, `installed_at=ranAt` (only if currently null), `last_ran_at=ranAt`. Else: only flip to `failed` if prior `status` was `attempting`; never downgrade `installed` to `failed`. Always update `last_ran_at`.
8. Return exitCode.

`ts_started` is captured at installer entry (before parse) and threaded through. Single-write model: every error path also writes one invocation row with both timestamps set.

### `src/store/db.ts`

```ts
import { Database } from "bun:sqlite";
import { sweepFs, ensureSweepHome } from "../fs.ts";

let db: Database | null = null;

/** Idempotent. Opens the DB (creating it if needed), enables WAL + FKs, runs migrations. */
export function ensureDb(): Database {
  if (db) return db;
  ensureSweepHome();   // bun:sqlite does not create parent dirs
  db = new Database(sweepFs.resolve("sweep.db"), { create: true });
  db.exec("PRAGMA journal_mode = WAL;");
  db.exec("PRAGMA foreign_keys = ON;");
  migrate(db);
  return db;
}
```

**Migrations.** Inline `CREATE TABLE IF NOT EXISTS` is sufficient for v0. Track schema version in a `schema_meta(version INTEGER UNIQUE)` row to enable real migrations later. v0 schema = version 1. `migrate()` runs `INSERT OR IGNORE INTO schema_meta (version) VALUES (1)` after the DDL so re-opens are idempotent.

### `src/store/packages.ts`

```ts
export type PackageRow = {
  id: number;
  slug: string;
  source_url: string;
  current_sha256: string | null;
  status: "attempting" | "installed" | "failed" | "uninstalled" | "tracked";
  first_seen_at: string;
  installed_at: string | null;
  last_ran_at: string | null;
};

export function findOrCreatePackage(args: { url: string; slug: string }): PackageRow;
export function listInstalledPackages(): PackageRow[];     // status='installed', ordered by installed_at DESC
// Encodes the status-transition rules from install.ts step 7. Reads the prior
// row, applies the transition based on exitCode + prior status, updates
// current_sha256 / installed_at (if null) / last_ran_at as appropriate.
// Caller wraps this and insertInvocation in one db.transaction().
export function updatePackageOnExec(args: {
  packageId: number;
  sha256: string;
  exitCode: number;
  ranAt: string;          // ISO 8601 — used for last_ran_at, and installed_at if newly installed
}): void;
```

`findOrCreatePackage` uses `INSERT … ON CONFLICT(source_url) DO NOTHING` then a follow-up `SELECT`. Returns the row whether it was just created or already existed.

`updatePackageOnExec` encodes the status-transition rules from `install.ts` step 7.

### `src/store/invocations.ts`

```ts
export type Outcome = "ran" | "cancelled" | "errored" | "parse_failed" | "fetch_failed";

export type Invocation = {
  id: string;                              // crypto.randomUUID()
  package_id: number | null;
  ts_started: string;                      // ISO 8601
  ts_finished: string | null;
  raw_input: string;
  url: string | null;
  sha256: string | null;
  install_command_json: string | null;
  outcome: Outcome;
  exit_code: number | null;
  error_message: string | null;
};

export function insertInvocation(e: Omit<Invocation, "id"> & { id?: string }): Invocation;
```

v0 inserts the invocation once at the end of the installer (in the same transaction as the package update). Crash-during-exec recovery (writing a "started" row before spawn) is deferred to v1. UUIDs come from `crypto.randomUUID()` — built-in, no dep.

### `src/store/scripts.ts`

```ts
export function scriptPath(sha256: string): string;                       // ~/.sweep/cache/scripts/<sha>
export function saveScript(sha256: string, bytes: Uint8Array): string;    // idempotent; returns absolute path
export function readScript(sha256: string): Uint8Array | null;
```

Path resolution goes through `sweepFs.resolve(...)`, but the bytes I/O uses `node:fs.writeFileSync` / `readFileSync` directly — `AppFs.write` is utf-8 text only (per `vault/wrap-core-api/fs.md`) and script bytes must round-trip exactly. `saveScript` also `mkdirSync`s the parent (`cache/scripts/`, recursive) and skips the write if the file already exists (same sha → same bytes by definition).

### `src/identity/naming.ts`

```ts
export function slugFromUrl(url: string): string;
```

v0 algorithm:

1. `new URL(url)`. If construction throws, return `"unknown"`.
2. Take `url.hostname.toLowerCase()`.
3. Strip a *single* leading prefix label if it matches one of: `www.`, `get.`, `install.`, `download.`, `dl.`, `cdn.`. At most one strip (no recursion).
4. Split on `.`, return the first label.
5. If the first label is empty (numeric IP, weird inputs), fall back to the full hostname.

Worked examples: `https://ollama.com/install.sh` → `ollama`. `https://get.bun.sh` → strip `get.` → `bun.sh` → `bun`. `https://mise.run` → `mise`. `https://install.foo.bar.com` → strip `install.` → `foo`.

v0 does not deduplicate slugs across packages; collisions are possible and tolerated (the `source_url` UNIQUE constraint disambiguates rows). `sweep list` shows the source URL alongside the slug so the user can tell colliding rows apart.

The function signature is stable across v0 and v1. v1 replaces the body with a registry call; callers don't change.

### `src/types.ts`

Single barrel for cross-module types: `InstallCommand`, `ParseError`, `FetchedScript`, `Outcome`, `Invocation`, `PackageRow`. Each module imports its types from here.

## Storage

### Filesystem

```
~/.sweep/                                 (or $SWEEP_HOME)
  sweep.db                                # SQLite, both tables
  cache/
    scripts/
      <sha256>                            # raw bytes, content-addressed
```

No JSONL log. No config file in v0.

### Schema (version 1)

```sql
CREATE TABLE IF NOT EXISTS schema_meta (
  version INTEGER NOT NULL
);
-- Insert version=1 on first init.

CREATE TABLE IF NOT EXISTS packages (
  id                    INTEGER PRIMARY KEY,
  slug                  TEXT NOT NULL,
  source_url            TEXT NOT NULL UNIQUE,
  current_sha256        TEXT,
  status                TEXT NOT NULL,    -- attempting | installed | failed | uninstalled | tracked
  first_seen_at         TEXT NOT NULL,
  installed_at          TEXT,
  last_ran_at           TEXT
);

CREATE TABLE IF NOT EXISTS invocations (
  id                    TEXT PRIMARY KEY,
  package_id            INTEGER REFERENCES packages(id),
  ts_started            TEXT NOT NULL,
  ts_finished           TEXT,
  raw_input             TEXT NOT NULL,
  url                   TEXT,
  sha256                TEXT,
  install_command_json  TEXT,
  outcome               TEXT NOT NULL,    -- ran | cancelled | errored | parse_failed | fetch_failed
  exit_code             INTEGER,
  error_message         TEXT
);

CREATE INDEX IF NOT EXISTS packages_sha   ON packages(current_sha256);
CREATE INDEX IF NOT EXISTS invocations_pkg ON invocations(package_id);
CREATE INDEX IF NOT EXISTS invocations_ts  ON invocations(ts_started DESC);
```

**Lifecycle of a `packages` row** (v0):

| Trigger | `status` after |
| --- | --- |
| First fetch for a `source_url` succeeds | `attempting` |
| Exec exits 0 | `installed` |
| Exec exits non-zero, prior status was `attempting` | `failed` |
| Exec exits non-zero, prior status was `installed` | unchanged (still `installed`; the new invocation records the failure) |

`uninstalled` and `tracked` are reserved for v1 (`sweep away`, `sweep track`) — never set in v0.

## Data flows

### `sweep "<curl … | sh>"`

```
argv ──▶ main.ts ──▶ installer/install.ts:runInstall(raw)
                                                  │
                                                  ├─▶ parse.ts          (→ ParseError → invocation, exit 2)
                                                  ├─▶ fetch.ts          (→ error → invocation, exit 1)
                                                  ├─▶ store/scripts.ts  (saveScript)
                                                  ├─▶ store/packages.ts (findOrCreatePackage)
                                                  ├─▶ identity/naming.ts
                                                  ├─▶ [v1 seam: analyze + confirm]
                                                  ├─▶ exec.ts           (run, capture exitCode)
                                                  └─▶ store/{invocations,packages}.ts (one txn)
                                                          │
                                                          └─▶ exitCode → main → process exit
```

### `sweep list`

```
argv ──▶ main.ts ──▶ subcommands/list.ts ──▶ store/packages.ts:listInstalledPackages()
                                                  │
                                                  └─▶ format + stdout
```

Output format (v0):

```
slug	source	installed	
ollama	https://ollama.com/install.sh	2026-05-20
bun	https://bun.sh/install	2026-05-20
mise	https://mise.run	2026-05-20
```

Plain text, no colors, no headers when empty. Empty list prints a single line: `No packages installed.` Exit 0. Unformatted timestamps.
